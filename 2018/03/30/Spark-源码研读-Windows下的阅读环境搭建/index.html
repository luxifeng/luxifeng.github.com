Spark | 源码研读 - Windows下的阅读环境搭建

Mar 30 2018

尽管在Linux下面编译Spark会更为方便，但是因为自己的机子上只安装了Windows以及懒得搭建虚拟机，因此索性尝试在Windows下面搭建Spark源码阅读环境，仅注意编译环境，其他无甚差别。

配置

  • os: Windows 10

  • java version: 1.8.0_144

  • spark version: 2.3.0

  • git version: 2.16.1.windows.1

  • maven version: Apache Maven 3.5.3

  • intellij idea version: 2017.3.5 社区版

下载及编译

  1. 下载Spark源码

  2. 编译Spark的原因

    为何阅读源码前要编译Spark？网上的解释是

    在将 Spark 工程导入 IDEA 之前，最好先 build 一遍 Spark，至少完成编译步骤。这是因为编译过程中还会自动生成一些源码。尤其是 Spark 2.0 中 SQL parser 改用 ANTLR 实现。如果不经编译直接将工程导入 IDEA，parser 相关的 class 会出现引用缺失。

    编译Spark的步骤跟着官方文档走即可，但在Windows环境下必须注意Linux Shell的执行问题，直接在Dos环境下执行编译会报错，使用Git客户端自带的工具MINGW64可行。

  3. Dos下编译Spark（失败！！！）

    期间下载各种jar包，等待等待~但是最后报错！！！

    Execute failed: java.io.IOException: Cannot run program “bash”

    就是说不能执行bash文件咯！恰好Git客户端自带了一个工具，可以执行Linux Shell命令，换Git Bash来！

  4. Git Bash下编译Spark（成功！！！）

  5. 导入IDE

    使用IntelliJ IDEA导入，默认配置即可。
